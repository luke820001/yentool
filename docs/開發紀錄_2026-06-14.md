## 一、專案概述

一套「可攜帶的 Windows 桌面 GUI 應用程式」，作為**台股籌碼爆發掃描器**。
核心目標：在全市場中，自動篩選出「正在低量盤整、主力默默吸籌、即將噴發」的潛伏股。

### 核心工程規範（持續遵守）
- **所有 `.py` 程式碼 100% 純 ASCII**：變數、函式、註解、字串一律英文。
- **GUI 顯示字串**：依需求改為中文（使用者明確要求）。
- **文件（如本檔）**：中文。
- **禁止推測外部 API**：實作前先列出資料集欄位 / API 簽名供確認。

## 二、系統架構（雙層漏斗）

```
[Step 1] 全市場快篩 (免費，無額度限制)
  TWSE OpenAPI + OTC OpenAPI
  -> 過濾 price < 100、排除 ETF(00 開頭)/DR
  -> 依成交量取 Top N (目前 N=50)
        |
[Step 2] 籌碼精算 (FinMind 免費 API)
  每檔抓 TaiwanStockPrice (近 90 天)
  -> 計算爆發分數 + 條件 A / C
        |
[Step 3] tkinter GUI 顯示
  -> 結果表格 + 壓力支撐欄位 + AI 報告按鈕
```

## 四、關鍵技術決策與踩坑紀錄

### 1. FinMind 免費方案限制
- **每小時約 300 次請求**。採「每檔僅抓增量」+「全市場快篩先縮減候選」雙重保護。

### 2. 大戶資料(條件 B)→ 停用
- `TaiwanStockShareholding` 實為「外資持股」，非股權分散表。
- 真正的千張大戶資料集 `TaiwanStockHoldingSharesPer` **需付費**（免費回傳 status 400）。
- **決策**：暫停條件 B，架構欄位保留，日後升級付費可無痛接回。

### 3. 券商分點(條件 C)→ 改用代理指標
- `TaiwanStockBrokerBranch` 免費方案**完全封鎖**（HTTP 422，與時間長短無關）。
- **決策**：改用「**量能偏多度 Volume Bias**」作為吸籌代理 —
  近 10 日「上漲日成交量 / 總成交量」，> 0.6 視為主力默默買進。

### 4. OTC OpenAPI 的 SSL 憑證問題
- `tpex.org.tw` 憑證不標準（Missing Subject Key Identifier）。
- **解法**：對 OTC 請求 `verify=False` 並關閉 urllib3 警告。

### 5. 爆發分數模型(0~100)
針對「近兩年飆股初期共同現象」設計三大加權指標：

| 指標 | 權重 | 意義 |
|---|---|---|
| 箱型壓縮度 Range_Tightness | 35 | 20 日高低差越窄分越高 |
| 量能萎縮比 Volume_Dryup | 35 | 當日量 / MA20 量，越低分越高 |
| 吸籌偏多度 Volume_Bias | 30 | 上漲日量佔比，> 0.6 才得分 |

觸發條件：
- **條件 A**：20 日箱型寬度 < 8% 且 當日量 < MA20 量的 60%
- **條件 C**：Volume_Bias >= 60%

### 6. 增量快取(加速關鍵)
- 本地 Excel 已有資料時，只抓「最後一筆之後」的新天數。
- 若已更新到今天，**完全跳過 API**，直接用快取重算指標。
- 效果：同日第二次掃描幾乎瞬間完成，且不耗 API 額度。

## 六、操作備忘

### 啟動 GUI
```powershell
cd D:\YenTool
python main_gui.py
```

### 流程
1. 按「開始掃描」→ 跑完後「產生 AI 報告」按鈕亮起
2. 第二次掃描會大量出現 `cache hit, skip API`(增量快取生效)
3. 換好 Gemini key 後，按「產生 AI 報告」彈出繁中分析

### 除錯工具
```powershell
python debug_api.py       # 探測 FinMind 資料集欄位
python debug_signals.py   # 檢視前 5 檔的訊號計算明細
```

### 重要設定位置(config/settings.py)
- `VOLUME_TOP_N`：掃描檔數上限
- `PRICE_FILTER_MAX`：股價上限(目前 100)
- `ROLLING_DAYS`：資料保留天數(目前 90)
- `GEMINI_MODEL`：目前 gemini-2.0-flash
